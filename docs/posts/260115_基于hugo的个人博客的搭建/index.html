基于hugo的个人博客的搭建

January 15, 2026
Words: 901 · Reading: 2 min
  • 探索

背景 / 目的（Why）

  • 想搭建个人观测系统，需要长期记录自己，构建一个个人站来统筹
  • hugo的博客基于本地文件，没有因为第三方商家停止服务后数据遗失的风险

数据 & 环境（Data & Setup）

  • 用到的工具包： choco git go npm
  • 方法：根据教程及其他用户的使用说明搭建

过程记录（What I Did）

  • 安装hugo，过程中熟悉了cmd的文件夹转移操作
  • 发现没有装git，同步安装了go
    • 程序员失格，之前github都是用的desktop git的
  • 在gpt指导下安装了papermod这个主题，但太简单
    • 没有前端设计经验，自己装修太难，于是打算装一个其他用户已搭建好的theme
  • 在clone其他用户theme的时候产生问题
    • 版本更新问题一些文档里语句已停止使用，需要修改
    • 第一个选择的theme诈骗，实装后图文不符
  • 实装theme后，设置了4个category的index，并设计了4个对应的文章模板
  • 原本模板中设置了logo栏位但没有说明引用路径，在sidebar相关scripts里找到后成功显示头像
  • 原本模板字体太大，页面信息量太少不适合电脑阅读，找到了设定字体的两个scripts并直接调整了设置
  • 发现在首页全部显示全文：
    • 一个原因是没有引入hasCJKLanguage而导致字数统计错误；
    • 另一个原因经过反复控制变量对比实验找到了：如果文本中有一些特殊情况分割，就不算进字数统计，.Truncated不会变成True。但目前还没找到具体是什么特殊情况。可能是中英混杂太多导致统计混乱。暂时还没解决，出现类似问题的文档会手动填写more语句

结果（Results）

  • 构建好了可使用的hugo blog，并有了一个较为美观的页面设计
  • 成功写了第一篇blog

分析 & 思考（Thoughts）

  • 有什么新发现： 发现我对前端确实一窍不通，而且因为gpt太快地把solution和code不加解释地灌输给我，实际上没有什么理解。但不是这次开发的重点

TODO / 下一步（Next）

  • 每天写journal固然好，idea需要随手记录，是否可能由手机端传输到电脑？手动输入太麻烦
  • 原本有将blog内容定期传到flomo进行自我洞察分析的打算。flomo的api和如何进行自动化数据传输还需要研究
